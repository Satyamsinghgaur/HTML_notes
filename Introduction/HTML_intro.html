Semantic Markup

It is the markup that relates to the "Meaning of content".
There are two types of tags:-

  • Semantic Tags:-

    We can guess the type of content by seeing the tag or to to guess meaning of tags.
    ie-heading tags, paragraph tag(p), image tag(img) etc.

  • Non-semantic Tags:-

    We can not guess the type of content by seeing the tag.
    ie-div tags, span tag etc.

Note :-

  • We try to write website mostly semantic so we can guess what kind of content that section have, for that we use "Section" tag, "Header" tag, "Footer" Tag instead of "Div" tag or "span" tag.

    Some semantic markups :-

    • Simgle use :-

    • Multiple use :-

      • "article" tag :-

        This is used to write some content. It is used to create a small document section containing "h1" to "h6" heading tags, images, links etc

      • "section" tag :-

        It is used to create a section like "div" tag, but it is semantic.

      • "aside" tag :-

        This is used to give "Indirect related content and links"

  • By using semantic tags the structure of the website become meaningful and structured.
  • Our Website become SEO friendly(Search engine optimization).
  • Website become readable and supportable for Screen Reader (Used by Blind People).
  • Improve UX (user experience).

    • HTML Entities

    • An Html entity is a piece of text ("String") that begins with an ampersand (&) and ends with a semicolon (;).
    • Used to display reserved characters (Which would otherwise be interpreted as HTML code), and invisible characters (like non-breaking spaces).
    • Can also use in place of characters that are difficult to type with a standard keyboard.
    • Browser interprets them and renders correct character.
